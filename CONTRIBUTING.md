# Contributing

Thanks for considering contributing to this starter. Keep changes small and focused.

How to contribute

- Fork the repo and open a pull request against `main`.
- Write a clear PR description explaining the change and why it helps service teams.
- Keep PRs small — one logical change per PR makes review easier.

Commit messages

- Prefer short, imperative messages. Example: `Add static-copy for govuk assets`.
- Optionally follow Conventional Commits if your team prefers them.

Local checks

- Install dependencies: `npm install` or `yarn`.
- Run the dev server: `npm run dev`.
- Build for production: `npm run build` and verify the `dist/` output.

Testing and quality

- This repo does not include automated tests by default. If you add tests, include instructions in this file.
- Run linters or formatters if added by the project.

Licensing

- By contributing, you agree that your contribution will be licensed under the repository `LICENSE`.

Code of conduct

- Be respectful and constructive. Follow the project's code of conduct if one is added.
